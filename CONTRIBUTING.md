Contributing to 
First off, thank you for considering contributing! All contributions, bug reports, bug fixes, documentation improvements, enhancements, and ideas are welcome.

To keep things simple and efficient, please follow these guidelines:

1. How Can I Contribute?
Reporting Bugs
Check the Issues tab to see if the bug has already been reported.

If not, create a new issue. Describe the problem, steps to reproduce, and your environment (e.g., Java version, OS).

Suggesting Enhancements
Open an issue with the tag enhancement.

Explain why this feature would be useful and how it should work.

Pull Requests (PRs)
Fork the repository and create your branch from main.

If you've added code that should be tested, add tests.

Ensure the code follows the project's style (Simplicity is key!).

Submit a pull request with a clear description of your changes.

2. Style Guidelines
Code: Keep it clean and readable. Use meaningful variable names (CamelCase for Java/JS).

Messages: Write clear commit messages (e.g., Fix: Resolve login timeout issue).

Documentation: Update the README.md if your changes affect how the project is used.
